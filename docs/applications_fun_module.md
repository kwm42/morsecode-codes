# Applications & Fun 模块设计文档

## 模块概览
“Applications & Fun（应用与趣味）”页面向用户展示摩斯密码在互动体验中的多样玩法，通过可视化演示、灯光模拟与轻量小游戏，帮助初学者与进阶玩家在真实或趣味场景中练习与应用摩斯密码。

---

## 设计目标
- **理解场景化应用**：用直观示例说明摩斯密码在求救、灯光通信与娱乐中的实际使用方式。
- **降低上手门槛**：提供清晰步骤与提示，帮助用户快速体验核心交互而不必掌握全部理论。
- **激发兴趣与分享**：通过小游戏与挑战鼓励用户在课堂、聚会或线上分享成果，提升平台趣味性。
- **保持品牌一致性**：延续站点明亮配色、圆角卡片与 Tailwind/Shadcn 样式体系，使用 next-intl 维护多语言文案。

---

## 模块细节

### 1. Header
- **标题、副标题**：引用 `applicationsFun.title` 与 `applicationsFun.description`。
- **辅助动作**：可提供“立即体验”或“返回学习页面”等按钮，采用圆角主色调按钮与次要描边按钮。
- **背景元素**：保持浅色渐变或抽象波纹，与 Learn 页面一致，避免喧宾夺主。

### 2. SOS Signal Demo
- **内容结构**：
  - 简介卡片：解释 SOS（三短三长三短）及其历史。
  - 控件区：播放/暂停、速度滑杆（WPM）、音调滑杆（Hz）。
  - 可视化：
    - 动态波形或点划序列动画。
    - 可选文字提示：`.` `-` 序列、字母 S/O/S。
- **交互状态**：
  - 初始为“待播放”状态，突出主按钮。
  - 提供播放完成提示，可引导用户重复练习。
- **无音频 fallback**：若浏览器不支持 Web Audio API，提示用户使用文本示例或下载音频。

### 3. Light Signal Demo
- **内容结构**：
  - 场景文案：介绍使用手电或屏幕闪烁发送摩斯的步骤。
  - 模拟组件：
    - 预设模式：基础闪烁（SOS）、自定义文本转闪烁（占位）。
    - 预览窗口：背景暗色，中央光点根据节奏亮灭。
  - 控制面板：
    - “开始模拟”按钮（占位）。
    - 亮度/颜色选择器。
    - 节奏说明（字符间、单词间停顿）。
- **可访问性**：提醒用户闪烁可能对部分人群不适，提供关闭提示选项。

### 4. Morse Puzzles & Mini-games
- **模块组成**：
  - 卡片集合展示 2–3 种玩法：
    1. **Pattern Puzzle**：拖拽点划片段组词。
    2. **Speed Typing Arena**：限时解码挑战。
    3. **Accuracy Challenge**：连续答题保持高准确率。
  - 每张卡片包含玩法简介、预计难度、Coming soon 提示。
- **扩展区域**：
  - 预留排行榜或分享链接区块。
  - 提示未来会整合账号系统以记录成绩。

### 5. Footer
- 复用全站 Footer，确保隐私政策、使用条款与社交链接可达。

## UI/风格规范
- **配色**：遵循 `src/app/theme.css` 定义的 `--primary`、`--secondary` 等变量。
- **排版**：使用 Tailwind 字体工具类，标题 `text-3xl` 起步，正文 `text-sm` 或 `text-base`。
- **组件**：
  - 卡片统一使用圆角 `rounded-3xl`、边框 `border-[var(--border)]`、背景 `bg-[var(--card)]`。
  - 按钮遵循 Shadcn 风格：主按钮填充 `bg-[var(--primary)]`，次按钮描边。
- **布局**：
  - 外层容器 `max-w-6xl`，`px-6` ～ `px-8`。
  - 模块之间保持 `gap-10` 或 `gap-12`。
  - 使用 `grid` 或 `flex` 构建响应式布局，手机端为一列。

## 文案与国际化
- 所有文案在 `src/i18n/messages/en.json` 与 `zh.json` 中维护，命名空间 `applicationsFun`。
- 若新增字段（如按钮标签、占位提示），需同步更新多语言文件。
- Coming soon 提示可沿用 Learn 页面现有键，或在此模块添加专属键值。

---

## 技术实现建议
- **音频播放**：使用 Web Audio API 或 Tone.js；保留 SSR 兼容性，必要时在客户端组件中调用。
- **动画**：可使用 CSS 动画、Framer Motion 或 Canvas；注意性能与可访问性。
- **状态管理**：轻量交互可在组件内部使用 `useState`；若需跨组件同步，使用现有 Context 方案。
- **无功能占位**：在功能未完成前，按钮显示“Coming soon”或禁用态，防止用户误解。
